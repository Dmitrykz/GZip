GZipTest 
Управление
• для архивации: GZipTest.exe compress [имя исходного файла] [имя архива]
• для разархивации: GZipTest.exe decompress  [имя архива] [имя распакованного файла]

В GZipTest  разаработана схема, которая распараллеливает архивирование (разархивирование) на 4 параллельных потока. В связи с условием того, что файлы могут быть намного больше оперативной памяти, то для параллельных потоков, создаются временые файлы.

Все действия происходят в классе Zip, объект которого создается в Main.
 
Основаной метод класса, который исполняется в Main - int Run(string[] args). Данный метод возвращет 1 если все прошло удачно и 0 если была ошибка.

в Run(string[] args) происходит выбор метода работы программы:

Для Архивирования. Метод -Comp() 
1)Четыре потока параллельно читают файл, сжимают информацию и записывают данные в 4 временных файла.  Метод -Compress()
2)Почередно данные из 2,3 и 4-го файлов записываются в первый, далее в конец записывается строка содержащая размеры, данных файлов соответсвенно. Данная информация будет необходима для разархивирования. Метод - CopyFileAfterComp()
3) 2,3 и 4 файлы удаляются. - CopyFileAfterComp() 

Для Разархивирования . Метод - Decomp() 
1)Файл разбивается на 4 части. Метод - PreFile()
2)Четыре потока параллельно читают полученные файлы, разархивировывают информацию и записывают данные в 4 временных файла,  разархивированные файлы сразу удаляются Метод- Decompress()
3)Почередно данные из 2,3 и 4-го файлов записываются в первый. CopyFileAfterDecomp()
4) 2,3 и 4 файлы удаляются.  CopyFileAfterDecomp()

Также для обеспечения работы и устранения ошибок, имеются следющие методы:
bool SetCommand(string[] args) - Инициализирует переменные при условии того, что информация из командной строки введена верно, возвращает true, если все прошло удачно
bool Check() - Проверяет, что информация из командной строки введена верно, возвращает true, если нет ошибок
 bool CrDirectory() -  Проверяет существует ли каталог в который создается новый файл, если нет то создает его.
bool FreeMemory() -Метод проверяет хватает ли места для выполнения программы

переменная iAns возвращает при помощи return либо 1 либо 0 в зависимотсти от результата работы.
